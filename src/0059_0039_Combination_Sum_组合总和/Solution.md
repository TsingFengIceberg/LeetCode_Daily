# [0039. 组合总和 (Combination Sum)]

## 💡 算法灵魂：无限弹药库的“防倒车”微调
这道题是经典的回溯“子集模板”的终极变种。当题目允许**“无限制重复选取同一个数字”**时，我们面临着既要“防倒车（避免 `[2,3]` 和 `[3,2]` 重复）”，又要“允许原地薅羊毛（允许 `[2,2]`）”的矛盾。
* **破局核心 (`start = i`)**：在进入下一层递归时，我们将起始指针依然设为当前的 `i`，而不是 `i + 1`。
* **物理意义**：由于下一层循环依然从 `i` 开始，它可以无限次地选中当前的数字；而一旦当前这层关于 `i` 的循环跑爆了（`target < 0`），触发了 `i++`，那么传递给更深层的起跑线就变成了 `i + 1`。跨过这个村，就彻底没了这个店，从物理上完美隔绝了重复组合的产生！

## 🏢 工业界真实工程实例：凑单、对账与一维下料
“给定候选集，允许无限次拿取，求完美拼凑”是真实的业务线里最核心的算法引擎：
1. **电商双十一精确凑单系统**：用户差 70 元凑满减，平台推荐单价 `[10, 20, 35]` 的商品。后台的促销引擎利用此回溯算法，算出 `[[35, 35], [20, 20, 20, 10]]` 等完美凑齐的方案推荐给用户，防止用户被平台“反薅”。
2. **企业 ERP 智能财务对账**：对公账户收到一笔 46000 元的无备注打款，财务利用此算法与公司 `[5000, 8000, 15000]` 的标准产品定价表进行组合碰撞，极速反推出客户可能购买的套餐组合，实现自动销账。
3. **制造业一维下料排产 (1D Cutting Stock)**：钢材厂将 10 米标准钢管切割成客户要求的 `[2, 3, 4]` 米短管。为了实现“零废料”（利润最大化），排产系统底层以此算法穷举所有完美切割组合，作为工业排布的前置数据。

> ### 👑 架构师的终极感悟：回溯 (DFS) 与 动态规划 (DP) 的抉择
> 在面试和真实架构中，遇到类似问题，**绝不能无脑写代码，必须先问清楚业务方的核心诉求！**
> 
> * **场景 A（只要结果和数量）**：如果业务方问：“能不能凑齐 70 元？” 或者 “凑齐 70 元总共有多少种方案？”
>   👉 **绝对不要用回溯！** 必须立刻切到**动态规划（完全背包问题）**！因为 DP 只记录状态和数量，不记录沿途的风景（路径），时间复杂度会被极致压缩到 $O(N \times \text{target})$。
> 
> * **场景 B（必须看具体清单）**：如果业务方要求（如本题）：“把所有能凑齐 70 元的**具体购买清单 / 组合路径**，原原本本地列出来给我看！”
>   👉 **没有任何捷径，只能用回溯 (DFS)！** 必须老老实实地用一套 `for` 循环深入每一个分支，走到尽头拍下快照（`new ArrayList<>(path)`），因为此时的业务价值在于**路径本身**。
> 
> **“知其然（会写代码），更知其所以然（懂业务取舍），才是 P7 级资深架构师的底色。”**

## 🧮 复杂度剖析
* **时间复杂度：$O(S)$**。其中 $S$ 为所有可行解的长度之和。最差情况下的时间复杂度是一个极其松散的上限 $O(N^{\frac{\text{target}}{\min}})$（`min` 为数组中最小的数字），因为每次最少减去 `min`，决策树最大深度为 $\frac{\text{target}}{\min}$。
* **空间复杂度：$O(\frac{\text{target}}{\min})$**。除答案数组外，空间复杂度取决于递归的栈深度。最坏情况下（每次都选最小的数字），递归深度可达 $\frac{\text{target}}{\min}$。

## 💻 最终 Java 代码实现 (大厂原生满分版)

```java
import java.util.List;
import java.util.ArrayList;

class Solution {
    List<List<Integer>> res = new ArrayList<>();
    
    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        // 可选优化（极致微操）：若在开头加上 Arrays.sort(candidates); 
        // 并在 for 循环内加上 if (target - nums[i] < 0) break; 
        // 可以实现极致剪枝，将耗时进一步压榨至 0ms-1ms。
        backtrack(candidates, new ArrayList<>(), target, 0);
        return res;
    }

    private void backtrack(int[] nums, List<Integer> path, int target, int start) {
        // 剪枝：废片处理（走爆了）
        if (target < 0) {
            return;
        }
        // 收集：成功快照（刚好凑齐）
        if (target == 0) {
            res.add(new ArrayList<>(path));
            return;
        }
        
        // 循环严格从 start 开始，物理隔绝 [3, 2] 的倒车可能
        for (int i = start; i < nums.length; i++) {
            path.add(nums[i]);
            
            // 🚨 魔法参数 i：允许无限次重复选取当前数字 nums[i]
            backtrack(nums, path, target - nums[i], i);
            
            // 撤销选择，恢复案发现场
            path.remove(path.size() - 1);
        }
    }
}
