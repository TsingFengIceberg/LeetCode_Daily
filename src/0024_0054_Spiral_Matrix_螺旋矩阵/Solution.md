# LeetCode 54. 螺旋矩阵 (Spiral Matrix) - 深度复盘笔记

## 💡 核心算法思想：剥洋葱（四面墙收缩法）
应对矩阵模拟路径问题，**摒弃追踪具体坐标点（易越界、难维护）**的思路，转而维护四个边界变量（`top`, `bottom`, `left`, `right`），将矩阵看作层层嵌套的洋葱，由外向内一层层剥离。

* **时间复杂度**：O(m * n)，其中 m 和 n 为矩阵的行列数，每个元素精确被访问一次。
* **空间复杂度**：O(1)，除用于返回结果的 List 外，仅使用了四个整型变量维护边界。

## 🧠 核心疑问与思维推导全纪录

### 疑问 1：为什么不能用“修改原数组加 300”的原地状态机标记法？
* **思考过程**：利用题目 `[-100, 100]` 的数据范围，走过的位置 `+300` 标记为已访问，配合方向数组 `[0, 1, 0, -1, 0]` 控制转向。
* **被否决原因（工程化视角）**：
  1. **魔数陷阱与普适性差**：高度依赖特定的数据范围约束，一旦业务变为真实金额或不限范围的 ID，立刻导致溢出或报错。
  2. **并发安全与脏数据**：在真实大厂后端环境，修改入参是一大忌讳。若该矩阵是多线程共享的缓存对象，中间状态的脏数据会导致严重的 P0 级线上事故。

### 疑问 2：为什么四向遍历中，前两个（向右、向下）不需要加 if 判断，而后两个（向左、向上）必须加？
* **逻辑盲区**：外层的 `while (top <= bottom && left <= right)` 只能在每一整圈（四个动作全做完）后才进行全局检查。
* **推导结论（破局点）**：
  * **向右走 / 向下走（自带防御）**：以“向下走”为例，它的循环条件是 `i <= bottom`。在第一步“向右走”完成后，`top` 执行了 `top++`。如果此时 `top` 已经越界大于了 `bottom`，那么“向下走”的 `for` 循环初始化条件 `top <= bottom` 直接不成立，**循环自带熔断机制**。
  * **向左走 / 向上走（盲目执行）**：以“向左走”为例，它的循环变量依赖的是 `left` 和 `right`。它**无法感知到**在第一步中 `top` 和 `bottom` 已经发生了交错（行被剥光了）。只要左右还有空间，它就会盲目扫荡导致“鞭尸”已读取的数据。因此，**必须手动加上雷达：`if (top <= bottom)`**。

## 🏭 工业界工程实例分析

跳出算法题，本题涉及的两套思维模型在真实大厂业务中有着极其广泛的应用：

1. **同心圆辐射搜索（边界扩张的逆向应用）**
   * **场景**：美团骑手调度 / 大众点评“搜索附近 3 公里的咖啡馆”。
   * **落地**：基于二维网格坐标（GeoHash），以用户为中心，利用 `top, bottom, left, right` 四面墙层层向外扩张去查库。找到了足够资源就立刻 `break`，完美避免了全表扫描，保护了数据库。
2. **状态机与方向数组的降维打击**
   * **场景**：阿里/字节《三国志·战略版》等 SLG 大地图游戏的军队网格寻路系统。
   * **落地**：将 `dirs = {0, 1, 0, -1, 0}` 封装为 Java Enum，结合 `(k + 1) % 4` 的状态机切换。用纯数据驱动代替几十行恶心的 `if-else` 转向判断，大幅降低代码圈复杂度。

## 💻 最终定稿 Java 代码 (大厂标准模板)

```java
import java.util.List;
import java.util.ArrayList;

class Solution {
    public List<Integer> spiralOrder(int[][] matrix) {
        // 1. 定义四面墙
        int top = 0, bottom = matrix.length - 1;
        int left = 0, right = matrix[0].length - 1;
        
        // 预分配容量，避免动态扩容带来的性能损耗
        List<Integer> result = new ArrayList<>(matrix.length * matrix[0].length);
        
        // 2. 只要四面墙没有交错，就继续向内剥洋葱
        while (top <= bottom && left <= right) {
            
            // 步骤一：向右走（贴上墙）
            for (int i = left; i <= right; i++) result.add(matrix[top][i]);
            top++; // 上墙下压
            
            // 步骤二：向下走（贴右墙）-- for循环自身带防御
            for (int i = top; i <= bottom; i++) result.add(matrix[i][right]);
            right--; // 右墙左推
            
            // 步骤三：向左走（贴下墙）-- 必须防碰撞
            if (top <= bottom) {
                for (int i = right; i >= left; i--) result.add(matrix[bottom][i]);
                bottom--; // 下墙上推
            }
            
            // 步骤四：向上走（贴左墙）-- 必须防碰撞
            if (left <= right) {
                for (int i = bottom; i >= top; i--) result.add(matrix[i][left]);
                left++; // 左墙右推
            }
        }
        
        return result;
    }
}
