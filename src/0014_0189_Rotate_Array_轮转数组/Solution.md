# 算法复盘笔记：LeetCode 189. 轮转数组 (Rotate Array)

## 一、 题目剖析与大厂考点
* **考察频次**：⭐⭐⭐⭐⭐ (字节跳动、阿里、腾讯等常考热身题)
* **核心标签**：数组、数学、双指针
* **时间/空间复杂度要求**：面试官终极期望为 Time: $O(N)$, Space: $O(1)$。
* **题目痛点**：看似简单，但要在 $O(1)$ 空间下完美处理所有的边界条件（如 $k > n$ 时的无效轮转，以及环状替换中的“死循环”陷阱），极度考验候选人代码的鲁棒性。

## 二、 核心解法沉淀

### 方法 1：三次翻转法 (最优解，强烈推荐手撕)
利用“局部翻转+整体翻转”的数学性质，将数组按轮转点切分。
* **思路**：
  1. 将整个数组翻转。
  2. 将前 $k$ 个元素翻转。
  3. 将剩余的 $n-k$ 个元素翻转。
* **复杂度**：时间 $O(N)$，空间 $O(1)$。
* **工程易错点 (Java)**：
  * Java 中原生数组 `int[]` **没有**自带的 `reverse` 方法，切忌强行转 `List` 调用 `Collections.reverse()`（会带来严重的装箱/拆箱开销）。
  * 必须手写基于 `while (start < end)` 的双指针 `reverse` 辅助方法。


**核心 Java 代码定稿**：
```java
public class Solution {
    public void rotate(int[] nums, int k) {
        int n = nums.length;
        k = k % n; // 剪枝与防越界
        if (k == 0 || n <= 1) return;

        reverse(nums, 0, n - 1);
        reverse(nums, 0, k - 1);
        reverse(nums, k, n - 1);
    }
    
    private void reverse(int[] nums, int start, int end){
        while (start < end){
            int temp = nums[start];
            nums[start] = nums[end];
            nums[end] = temp;
            start++;
            end--;
        }
    }
}

```

### 方法 2：环状替换法 (极致炫技，高难度)

像“抢座游戏”一样，直接把元素放到最终的计算位置 `(current + k) % n`。

* **复杂度**：时间 $O(N)$，空间 $O(1)$。
* **难点**：当 $n$ 和 $k$ 最大公约数大于 1 时，会产生“闭环”（鬼打墙），导致部分元素未被遍历。
* **破局方案**：引入 `count` 记录成功交换的元素个数，引入外层循环 `start++` 在陷入死循环时强行拨动指针开启新环。

### 方法 3 & 4：空间换时间 & 暴力冒泡

* **额外数组法**：开辟新数组 `newArr[(i + k) % n] = nums[i]`，再用高性能的 `System.arraycopy` 拷回。空间 $O(N)$。业务代码常用，面试非最优。
* **暴力冒泡法**：每次挪动 1 步，挪动 $k$ 次。时间 $O(N \times K)$，在大厂机试中必报 Time Limit Exceeded (TLE)。

## 三、 大厂实战变形与 Follow-up（面试真题）

在真实的字节/阿里后端面试中，这道题通常会披着其他外衣出现：

1. **数据结构变种：如果是链表怎么办？**
* **对应题目**：LeetCode 61. 旋转链表 (Rotate List)
* **解法差异**：链表不需要翻转，直接将链表首尾相连成环，然后在第 $n - k$ 个节点处断开即可！


2. **字符串变种：左旋转字符串 / 翻转字符串里的单词**
* **对应题目**：LeetCode 151. 反转字符串中的单词 / 剑指 Offer 58. 左旋转字符串
* **解法思想**：依然是核心的“整体翻转 + 局部翻转”套路。


3. **架构/工程映射**：
* 在高性能队列（如 LMAX Disruptor）和定时任务调度（Netty Hashed TimeWheel）中，均使用了“数组+取模”的环状设计来避免元素的物理拷贝，将 $O(N)$ 甚至 $O(\log N)$ 的操作降维至 $O(1)$。
