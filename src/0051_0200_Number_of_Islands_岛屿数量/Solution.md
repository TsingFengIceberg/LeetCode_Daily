# [0200. 岛屿数量 (Number of Islands)]

## 💡 算法战术选择：为什么大厂白板手撕偏爱 DFS？
面对二维网格的连通性问题（如寻找岛屿），BFS（广度优先）和 DFS（深度优先）在理论上均可完美解题。但在真实的大厂面试环境下，**DFS 是绝对的首选**。
1. **代码极简与直觉映射**：DFS 凭借“不撞南墙不回头”的特性，只需寥寥数行递归代码即可完成探索。而 BFS 需要手动引入队列（`Queue`）、频繁实例化坐标对象（如 `new int[]{x, y}`），在紧张的面试中不仅大幅增加代码量，还极易写错边界。
2. **复杂度剖析**：无论 DFS 还是 BFS，这套打法的时间复杂度均为 $O(M \times N)$（完整扫描网格 + 常数次递归探测）。但必须注意，面试官极爱设陷阱的空间复杂度同样也是 $O(M \times N)$！在最坏情况下（整个网格全是一整块陆地），DFS 的调用栈深度会达到网格格子的总数。

## ☕ 状态机设计与大厂“沉岛策略” (In-place Modification)
在探讨如何标记已访问过的陆地时，我们避免了极其臃肿的 $O(M \times N)$ 额外空间 `boolean[][] visited` 数组。
通过敏锐的工程直觉，我们采用了**“状态原地修改（沉岛战术）”**：
一旦外层雷达扫描到新大陆 `'1'`，立刻指派 DFS 登陆部队将这块陆地及所有相连的陆地全部“炸沉”——即原地修改为 `'0'`（海水）。这样一来，外层双重 `for` 循环扫描到这些格子时，会自然将其当作普通海水跳过，兵不血刃地解决了“重复统计”的痛点。

## 🏢 工业界真实工程实例：连通块挖掘与分布式痛点
这道算法题的本质是图论中的**“连通块（Connected Component）挖掘”**。在亿级并发的现代后端架构中无处不在：
1. **风控系统的黑灰产团伙挖掘（阿里/腾讯）**：将用户视为节点，相同的 IP/设备指纹视为相邻的 `'1'`。顺藤摸瓜找出的每一个“岛屿”，就是一个具备高度协同欺诈嫌疑的黑产团伙，系统依此进行批量封号。
2. **LBS 蜂窝网格与热区调度（美团/滴滴）**：将城市划分为巨大的六边形网格，订单量激增的网格设为 `'1'`。算法找出的“岛屿”即为当前城市互不相连的“爆单热区”，调度系统直接向岛屿中心倾斜运力。
3. **架构师进阶（防栈溢出与分布式运算）**：在单机场景下，递归 DFS 十分优雅。但在真实工程中，如果“岛屿”面积以亿计，几亿层递归会直接引爆 JVM 产生 `StackOverflowError`。此时大厂会采用 **手动显式队列（BFS）** 防止栈溢出；如果图谱庞大到单机内存无法装下，则会跨入大数据的范畴，使用 **Spark GraphX 分布式并查集 (Union-Find)** 来聚合计算岛屿数量。

## 💻 最终 Java 代码实现

```java
class Solution {
    public int numIslands(char[][] grid) {
        if (grid == null || grid.length == 0) {
            return 0;
        }
        
        int count = 0;
        int rows = grid.length;
        int cols = grid[0].length;
        
        // 1. 外层雷达扫描：遍历整个二维网格
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                // 如果发现了新大陆 '1'
                if (grid[i][j] == '1') {
                    // 岛屿数量加 1
                    count++;
                    // 派谴登陆部队，去把这座岛连着的陆地全部“沉掉”！
                    dfs(grid, i, j);
                }
            }
        }
        return count;
    }
    
    // 2. 核心递归函数：沉岛战术
    private void dfs(char[][] grid, int r, int c) {
        // 【第一步：写递归的终止条件 (Base Case)】
        // 核心防线：坐标越界，或是遇到海水 '0'，登陆部队立刻撤退
        if (r < 0 || r >= grid.length || c < 0 || c >= grid[0].length || grid[r][c] == '0') {
            return; 
        }
        
        // 【第二步：执行沉岛】
        // 原地修改状态：把当前这块陆地变成海水 '0'，防止外层循环重复统计
        grid[r][c] = '0'; 
        
        // 【第三步：四面八方蔓延】
        // 向 上、下、左、右 四个方向继续发起 DFS 进攻
        dfs(grid, r - 1, c); // 向上
        dfs(grid, r + 1, c); // 向下
        dfs(grid, r, c - 1); // 向左
        dfs(grid, r, c + 1); // 向右
    }
}
