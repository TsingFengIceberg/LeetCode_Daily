# [0236. 二叉树的最近公共祖先]

## 🚨 警惕指数级递归陷阱：从 TLE 到 $O(N)$ 的进化
在最初尝试手写递归时，你直觉性地写出了向上汇报的逻辑，但遭遇了 **TLE（超时）**。根本原因在于**疯狂的重复计算**：在同一个父节点下，你为了判断并返回结果，连续多次调用了 `lowestCommonAncestor(root.left, p, q)`。
这在树形结构中是极其致命的，它会让原本 $O(N)$ 的遍历时间复杂度瞬间膨胀为指数级的 $O(2^N)$。大厂面试中，**“将递归探查的结果用变量暂存起来”**是绝对的底线要求。通过声明 `TreeNode left = ...` 和 `TreeNode right = ...`，我们保证了整棵树中的每个节点最多只被访问一次，成功将时间复杂度降维打击回 $O(N)$。

## 💡 灵魂三问与后序遍历的汇报机制
本题是经典的二叉树**后序遍历（自底向上回溯）**的绝佳应用。它的核心思想是“先让下属去查，汇总下属的报告后，再向我的上级汇报”。
- **Base Case 剪枝**：如果当前节点 `root` 刚好就是被寻找的 `p` 或 `q`，完全没必要再去它的子树里找了（因为哪怕另一个节点在它下面，它自己也就是 LCA 了），直接向上级原路返回 `root` 本身。
- **汇总逻辑（核心）**：
  1. 如果左右小弟都交上了非空的人（`left != null && right != null`），说明 `p` 和 `q` 分别盘踞在我的左右两边，那我本人必然就是“最近公共祖先”，直接向上级汇报我 `root`。
  2. 如果只有一边交了人，说明 `p` 和 `q` 都在那一边，那我直接当个传话筒，把那边交上来的人继续向上传递（`return left != null ? left : right`）。

## 🏭 工业界真实工程实例：微服务排障与 OA 审批流
虽然业务代码里很少写纯粹的二叉树遍历，但 LCA 算法的思想在大厂真实高并发/分布式架构中无处不在，常见变种包括但不限于：
1. **分布式微服务全链路追踪（Trace Tree 排障）**：当系统大盘显示底层节点 `p`（如库存服务）和节点 `q`（如积分服务）同时大面积超时报警，底层监控系统（如 SkyWalking / 阿里鹰眼）会迅速跑一遍多叉树的 LCA 算法，找到调用这两个分支的**共同上游源头服务**，并将其标红，实现故障根因（Root Cause）的秒级自动化定位。
2. **企业级 OA 审批流决策（RBAC 动态寻址）**：在钉钉或飞书的数万人组织架构树中，当节点 `p`（某业务线底层研发）需要申请调用节点 `q`（跨部门核心资产）的权限时，系统会自动利用 LCA 计算出同时管辖这俩人的“最低级别共同高管”，并将审批单精准推送到该高管侧，彻底消灭人工配置审批流的繁琐。

## 最终 Java 代码实现
> *注：本次优化后的代码执行用时 7ms（击败 99.99%），彻底清除了冗余计算，每个节点仅遍历一次；内存消耗 45.86MB（击败 27.10%），符合标准 $O(N)$ 的系统调用栈深度。这是一份可以直接在字节/阿里全票通过的满分代码。*

```java
class Solution {
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        // 1. Base Case：如果碰到底了，或者遇到了 p 甚至遇到了 q，直接向上返回 root（剪枝优化）
        if (root == null || root == p || root == q) {
            return root;
        }
        
        // 2. 派左小弟去左子树找，【严禁重复调用，必须用变量存下结果！】
        TreeNode left = lowestCommonAncestor(root.left, p, q);
        // 3. 派右小弟去右子树找，【同样用变量存下结果！】
        TreeNode right = lowestCommonAncestor(root.right, p, q);
        
        // 4. 汇总左右小弟的汇报结果 (灵魂三问)
        if (left != null && right != null) {
            // 左右都找到了人，说明 p 和 q 分散在我的两边，那我 root 就是最近公共祖先！
            return root; 
        }
        
        // 如果左边为空，说明都在右边，把右边的结果汇报给上一级
        // 如果右边为空，说明都在左边，把左边的结果汇报给上一级
        return left != null ? left : right;
    }
}
