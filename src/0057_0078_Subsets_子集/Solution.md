# [0078. 子集 (Subsets)]

## 💡 算法灵魂：回溯模板的“降维微调”
“子集”与“全排列”是回溯算法的两座双子塔，理解它们的差异是掌握 DFS 的关键：
1. **收集时机（无条件拍照）**：排列必须等到数字凑齐才能存入结果；而子集的决策树上，**每一个节点都是合法结果**。因此，进入回溯函数的第一步，就是无条件执行 `res.add(new ArrayList<>(path))` 拍下快照。
2. **防倒车机制（废弃 `used` 数组）**：子集不讲究顺序（`[1,2]` 和 `[2,1]` 视为同一种）。为了防止重复，我们引入了 `start` 指针，规定每次 `for` 循环只能从当前元素的**下一个位置**开始挑。这个“绝不回头”的铁律，让全排列中臃肿的 `used` 状态数组直接下岗。

## 🚀 架构师深潜：工业界如何对抗 $O(2^N)$ 的指数爆炸？
在算法题中，数组长度 $N \le 10$，回溯跑得飞快。但在大厂真实的“商品智能打包”或“AI 推荐系统特征选择”中，$N$ 往往是几十甚至成千上万。当 $N=40$ 时，子集数量高达 **1 万亿**，纯靠穷举的服务器会当场熔断。面对这种 $O(2^N)$ 的灾难，真实的工业界有以下三套保命方案：

### 1. 业务规则的“物理隔绝” (Heuristic Pruning / 启发式剪枝)
在美团的套餐生成业务中，绝不进行无脑穷举。
* **规则剪枝**：如果一个子集（套餐）的单品数量已经超过 5 个，直接 `return` 砍掉整棵子树（用户根本吃不完）。
* **成本剪枝**：如果在递归途中，发现当前子集里的商品总成本已经超过了系统设定的 50 元上限，即使还没选完，也立刻 `break` 止损。
* **效果**：通过强业务规则注入回溯算法，能把 $O(2^N)$ 的搜索空间强行压缩到万分之一。

### 2. 生物学的降维打击 (遗传算法 Genetic Algorithms)
在面临 $N=100$ 的特征组合问题时，穷举已彻底失效。大厂会引入“进化论”。

* **基因编码**：把一个特征子集看作一段 DNA 序列，选了就是 1，没选就是 0。比如 `[1, 0, 1]`。
* **适者生存**：随机生成 1000 个子集（种群），让它们去跑模型。准确率高的子集被保留下来，互相**“交配 (Crossover)”**（比如把前一半特征和后一半特征拼接），并引入极小概率的**“变异 (Mutation)”**（随机选一个新特征）。
* **效果**：不需要遍历 $2^{100}$ 次，只要繁衍几十代，就能极速收敛到一个极其接近绝对最优解的“次优子集”。

### 3. 高阶数学的降维 (L1 正则化 / Lasso 惩罚)
在字节跳动的核心推荐算法底层，为了找出最优特征子集，AI 工程师甚至连“搜索”这个动作都省了。
* 算法模型在训练时，直接把所有 100 个特征全扔进去，但在数学公式（损失函数）里加上一个 **L1 惩罚项**。
* 这个惩罚项的物理意义是：强迫模型在梯度下降的过程中，把那些没用的、冗余的特征的权重，**绝对地压缩成 0**。
* **效果**：训练结束时，权重不为 0 的特征，天然就组成了一个最完美的特征子集！将离散的组合爆炸问题，化解为了连续的微积分求导问题。

## 🧮 复杂度剖析 (LeetCode 标准)
* **时间复杂度：$O(N \times 2^N)$**。集合共有 $2^N$ 种可能（每个元素选或不选）。对于每一种子集，我们需要花费 $O(N)$ 的时间通过 `new ArrayList<>()` 将其深拷贝到最终结果集中。
* **空间复杂度：$O(N)$**。递归的最大深度为 $N$，用来记录当前路径的 `sub` 列表最大长度也为 $N$。所以总辅助空间复杂度是线性的。（注：返回结果占用的 $O(N \times 2^N)$ 空间按惯例不计入空间复杂度评估）。

## 💻 最终 Java 代码实现 (大厂回溯防倒车版)

```java
import java.util.List;
import java.util.ArrayList;

class Solution {
    // 全局结果集，相当于存放所有快照的档案室
    List<List<Integer>> res = new ArrayList<>();
    
    public List<List<Integer>> subsets(int[] nums) {
        // 起手式：传入原数组、空的当前子集 sub（作为初始路径）、以及控制遍历不回头的 start 指针
        backtrack(nums, new ArrayList<>(), 0);
        return res;
    }

    private void backtrack(int[] nums, List<Integer> sub, int start) {
        // 【核心差异 1：无条件拍照收集】
        // 决策树的【每一个节点】都是合法答案（包括空集），因此一进函数立刻拍快照存入结果集！
        res.add(new ArrayList<>(sub));

        // 【核心差异 2：防倒车的 start 指针】
        // 循环严格从 start 开始，保证选过 nums[i] 后，下一层只能选它后面的元素。
        // 这彻底杜绝了集合顺序重复，也淘汰了全排列中的 used 数组！
        for (int i = start; i < nums.length; i++) {
            // 做选择：抓住当前的数字 nums[i]，加入子集
            sub.add(nums[i]);
            
            // 递归进入下一层：🚨 极其关键，传入 i + 1！永远不回头！
            backtrack(nums, sub, i + 1);
            
            // 撤销选择 (回溯/悔棋)：把刚加进去的元素踢出来，恢复案发现场
            sub.remove(sub.size() - 1);
        }
    }
}
