# [0074. 搜索二维矩阵 (Search a 2D Matrix)]

## 💡 算法灵魂：虚拟降维打击与坐标映射
面对严格递增的二维矩阵，平庸的做法是开辟新数组把元素全搬过去再二分，白白浪费 $O(M \times N)$ 的内存。
大厂的绝对标准解法是**“虚拟降维”**：我们在逻辑上把矩阵看作一个长条的一维数组，但在物理内存中依然保持它的二维形态。全题的灵魂在于**一维与二维空间的坐标系反向推导**。



* **起跑线与终点**：矩阵共有 $m \times n$ 个元素，所以一维逻辑的右边界直接设为 `m * n - 1`。
* **神仙坐标映射**：算出逻辑中点 `mid` 后，如何去真实的二维矩阵里拿数据？
  * **行号 (`mid_i`)** = `mid / 列数` （看前面排满了多少个完整的行）。
  * **列号 (`mid_j`)** = `mid % 列数` （看在当前这行还多出几个身位）。
* 仅仅依靠这两行小学水平的除法和取模运算，就完美实现了 $O(1)$ 空间复杂度的降维打击！

## 🏢 工业界真实工程实例：内存寻址与海量数据路由
这种“一维/二维虚拟坐标映射”算法，在底层系统开发中无处不在：
1. **数据库分库分表 (Sharding) 中间件**：当百亿级单表被切分成上千张小物理表时（构成巨大的二维矩阵），中间件（如 ShardingSphere）通过类似的除法和取模逻辑，瞬间算出你要找的 User ID 落在第几个物理表（行）的第几个位置（列），实现海量数据的极速路由。
2. **AI 框架 (TensorFlow / PyTorch) 张量内存寻址**：无论你在代码里操作多么复杂的三维、四维张量，物理内存条永远是一维线性的。底层 C++ 引擎正是依赖这种 `偏移量 = 行 * 列数 + 列` 的映射公式，在不发生真实数据搬运的情况下，完成极速的张量矩阵运算。
3. **大数据列式存储 (Parquet) 稀疏索引**：在 PB 级 Hadoop 集群中，排好序的数据块拼成逻辑上的巨型矩阵。查询引擎通过二分块边界索引，瞬间定位目标时间戳落在哪个具体的 Block（行）中，极大地避免了无效的磁盘 I/O 扫描。

## 🧮 复杂度剖析
* **时间复杂度：$O(\log(m \times n))$**。由于矩阵总元素个数为 $m \times n$，我们对全局进行了一次标准的二分查找，每次砍掉一半的搜索空间。等同于 $O(\log m + \log n)$。
* **空间复杂度：$O(1)$**。全过程只维护了 `left`, `right`, `mid`, `mid_i`, `mid_j` 这几个整型变量，没有任何额外的内存开辟，空间利用率拉满。

## 💻 最终 Java 代码实现 (0ms 大厂虚拟降维版)

```java
class Solution {
    public boolean searchMatrix(int[][] matrix, int target) {
        // 1. 设置一维虚拟数组的起跑线
        int left = 0;
        // 逻辑上的最大下标：总元素个数 - 1
        int right = matrix.length * matrix[0].length - 1;
        
        // 2. 经典的闭区间二分查找引擎
        while (left <= right) {
            // 防溢出计算虚拟中点
            int mid = left + (right - left) / 2;
            
            // 🚨 降维打击的核心魔法：一维下标转二维物理坐标
            // (注：在大厂极致微操中，可提前用 int cols = matrix[0].length; 提取列数，减少循环内的数组读取开销)
            int mid_i = mid / matrix[0].length; // 所在的行号
            int mid_j = mid % matrix[0].length; // 所在的列号
            
            // 拿到真实的矩阵元素，后续完全复用一维二分逻辑
            if (matrix[mid_i][mid_j] == target) {
                return true;
            } else if (matrix[mid_i][mid_j] < target) {
                left = mid + 1; // 目标值在右侧，收缩左边界
            } else {
                right = mid - 1; // 目标值在左侧，收缩右边界
            }
        }
        
        // 循环结束仍未找到，返回 false
        return false;
    }
}
