# [0300. 最长递增子序列 (Longest Increasing Subsequence)]

## 💡 算法灵魂与讨论实录：从 DP 陷阱到二分降维打击
本题是子序列问题的绝对核心。我们经历了从 $O(n^2)$ 动态规划到 $O(n \log n)$ 贪心二分的两次认知跃迁，期间通过极端的测试用例彻底拆解了算法的物理错觉。

### 1. 动态规划的“状态定义”陷阱与全局 Max
* **疑问**：在 DP 解法中，内层循环已经有了 `dp[i] = Math.max(dp[i], dp[j] + 1);`，为什么外层还需要一个全局的 `max = Math.max(max, dp[i]);`？最大长度难道不是随着 `i` 往后越来越大吗？
* **真相实录**：`dp[i]` 的绝对物理意义是“**必须以 `nums[i]` 为结尾**的最长长度”。最长序列不一定长在数组最后！
* **推演例子**：以 `nums = [10, 20, 30, 1]` 为例。走到 `30` 时，`dp[2] = 3`。但走到最后的 `1` 时，它谁都接不上，只能自己单干，导致 `dp[3] = 1`。如果没有全局的 `max` 记录下曾经的辉煌 `3`，最终结果就会错误地返回 `1`。外层的 `max` 就像是全国总决赛的星探，负责在所有“必须以某元素结尾”的平行宇宙中，挑出真正的总冠军。

### 2. $O(n \log n)$ 贪心+二分的“狸猫换太子” (Patience Sorting)
大厂进阶要求利用“维护 `tails` 数组 + 二分查找”将时间复杂度压缩至 $O(n \log n)$。`tails[i]` 代表：**长度为 `i+1` 的所有递增子序列中，结尾最小的那一个数字（越小潜力越大）**。



我们在讨论中，用三个极其精妙的极端例子，彻底扒光了 `tails` 数组的底裤：

* **案例一：`nums = [2, 3, 5, 6, 4, 7]` (内部换血不影响大局)**
  当走到 `4` 时，`tails` 是 `[2, 3, 5, 6]`。`4` 无法追加，于是通过二分查找替换掉了 `5`，变成 `[2, 3, 4, 6]`。虽然原数组中 `4` 在 `6` 后面，物理上不存在这个序列，但它代表系统发现了一个更有潜力的长度为 3 的序列（结尾是 4）。随后 `7` 到来，顺利追加，最终长度依然是精准的 5。
  
* **案例二：`nums = [2, 3, 5, 6, 4]` (戛然而止的备胎)**
  和案例一相同，最后一步 `4` 替换了 `5` 变成 `[2, 3, 4, 6]`，然后数组结束。虽然备胎 `4` 没派上用场，但这丝毫不会影响我们已经记录在案的历史最高长度 `len = 4`。

* **案例三：`nums = [1, 2, 11, 12, 5, 6, 7, 8, 9]` (绝地大翻盘)**
  初期高歌猛进形成僵尸序列 `[1, 2, 11, 12]`（长度 4）。随后神兵天降，`5` 替换 `11`，`6` 替换 `12`，在不损失当前最大长度的前提下，完成了内部核心的热更新重构，变成 `[1, 2, 5, 6]`！这为后面接纳 `7, 8, 9` 铺平了通天大道，最终精准测出长度为 `7`！

* **🏆 最终顿悟（原话记录）**：“所以 tail 在某种程度上只能代表结果的长度，而不能代表具体结果是吧？”
  **完全正确！** `tails` 数组是一个物理上可能根本不存在的“缝合怪”（例如 `nums = [4, 5, 6, 3]` 会得出 `tails = [3, 5, 6]`）。它唯一的使命，就是**精准记录最长序列的“刻度（长度）”！**

## 🏢 工业界真实工程实例：从代码合并到物种进化
1. **Git 的底层代码比对 (Diff 算法)**：找两个十万行文件的最小修改步骤，本质是求最长公共子序列 (LCS)。科学家证明 LCS 可以转换为求另一方的最长递增子序列 (LIS)。Git 底层正是利用了这套 $O(n \log n)$ 的降维算法，在毫秒级内完成极速的代码合并与冲突检测。
2. **生物信息学与基因组测序**：比对物种基因时，测序软件用 LIS 算法过滤掉亿万年进化中的基因突变和倒置“噪音”，找出编号稳定递增的最长“同源基因块 (Synteny Block)”，从而证明物种进化树。
3. **高铁站台分配 (Dilworth 定理)**：离散数学中，将列车进出站时间转化为偏序数组求 LIS，其长度就是为了避免列车冲突，物理上“最少需要修建的站台数量”。

## 🧮 复杂度剖析
* **时间复杂度：$O(n \log n)$**。遍历数组耗时 $O(n)$，每次遍历中执行二分查找耗时 $O(\log n)$，整体时间复杂度被完美压缩，属于性能怪兽级别。
* **空间复杂度：$O(n)$**。开辟了 `dp` 数组（解法一）或 `tail` 数组（解法二），占用线性空间。

## 💻 最终 Java 代码实现 (双修版)

### 解法一：动态规划 (保底基本盘 O(n²))
```java
class Solution1 {
    public int lengthOfLIS(int[] nums) {
        int n = nums.length;
        if (n < 2) {
            return n;
        }
        // dp[i]: 必须以 nums[i] 为结尾的最长递增子序列长度
        int [] dp = new int[n];
        dp [0] = 1;
        int max = 1; // 全局星探，记录历史最高分
        
        for (int i = 1; i < n; i++) {
            dp [i] = 1;
            // 内部竞争：找前面能接上去的最优“大哥”
            for (int j = 0; j < i; j++) {
                if (nums [j] < nums [i]) {
                    dp [i] = Math.max(dp [i], dp [j] + 1);
                }
            }
            // 全国总决赛：当前选手的成绩与全局历史最高分比对
            max = Math.max(max, dp [i]);
        }
        return max;
    }
}
```

### 解法二：贪心 + 二分查找 (2ms 降维打击 O(n log n))
```java
class Solution2 {
    public int lengthOfLIS(int[] nums) {
       // tail 数组：长度为 i+1 的递增子序列中，结尾最小的数字（缝合怪尺子）
       int [] tail = new int [nums.length];
       int size = 0; // tail 的有效长度
       
       for (int x : nums) {
           // 二分查找模板：在 [0, size) 区间找 x 的插入/替换位置
           int i = 0, j = size;
           while (i != j) {
               int m = (i + j) / 2;
               if (tail [m] < x) {
                   i = m + 1; // 逼近右侧
               } else {
                   j = m;     // 逼近左侧，找第一个大于等于 x 的
               }
           }
           // 无论如何，强行上位（潜伏替换备胎，或追加）
           tail [i] = x;
           // 如果 x 比前面的都大，成功开疆扩土，长度破纪录！
           if (i == size) {
               size++;
           }
       }
        return size;
    }
}
