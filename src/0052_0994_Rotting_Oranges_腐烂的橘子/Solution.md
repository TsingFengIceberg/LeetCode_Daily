# [0994. 腐烂的橘子 (Rotting Oranges)]

## 💡 战术抉择：为什么果断抛弃 DFS 选择 BFS？
在处理网格连通性时，DFS（深度优先搜索）是一把好手。但本题的核心物理定律是**“每分钟按层级、波纹状、同时向外扩散”**。如果使用 DFS，一条路走到黑的特性会瞬间让十万八千里外的橘子在“0毫秒”内腐烂，导致时间计算彻底混乱。
为了完美模拟这种随时间推移的“涟漪扩散”效应，**BFS（广度优先搜索）**天然自带的**按层扩展**属性是这道题的唯一真神。

## 🚀 核心大杀器：多源 BFS (超级源点) 与 O(1) 终局审判
大厂面试中这道题的两个核心考察点：
1. **多源并发（超级源点）的降维打击**：第 0 分钟可能同时存在多个烂橘子。新手常犯的错误是对每个起点单独跑一次 BFS，导致复杂度飙升至 $O(K \times M \times N)$。正确的工程做法是：**在初始化阶段，将所有烂橘子的坐标一次性全部推入队列！** 这样它们就会在第一分钟“同时”向外引爆，将复杂度极其优雅地降维回纯线性的 $O(M \times N)$。
2. **`count` 计数器法实现 O(1) 终局判定**：如何高效判断最后是否还有橘子烂不掉（返回 `-1`）？不需要最后再写两层 `for` 循环去扫网格。在最开始遍历时，顺手统计新鲜橘子的总数 `count`。每成功感染一个，`count--`。扩散结束后，只需极速判定 `count == 0` 即可。

## ⚡ 性能调优微操：为什么手动展开 4 个 if 击败了 100%？
在常规的大厂代码规范中，向四个方向扩散通常会定义一个 `int[][] dirs = {{-1,0}, {1,0}, {0,-1}, {0,1}}`，然后用 `for` 循环遍历。
但在追求极致性能的 LeetCode 环境中（也是底层中间件源码的常见微操），**直接手动展开写 4 个并列的 `if` 语句，省去了内部小循环的变量分配、压栈出栈和数组寻址开销。** 结合 `while (!queue.isEmpty() && count > 0)` 的精准剪枝，最终跑出了 **1ms (击败 100%)** 的极限物理性能！

## 🏢 工业界真实工程实例：微服务雪崩预演与 CDN 边缘推送
这套“多源 BFS”代码骨架，在分布式架构中大有可为：
1. **微服务架构中的“故障爆炸半径”分析**：当机房 3 个核心数据库节点同时宕机（初始烂橘子），监控系统（如 Prometheus）会立刻运行多源 BFS，按分钟层级预测接下来会被拖垮的上游微服务，赶在雪崩前精准下发“熔断降级”指令。
2. **CDN 与社交网络的病毒式推送**：当遇到全民热点事件，腾讯/字节会将全量数据先下发给全国 10 个骨干网节点（超级源点），然后利用多源 BFS 模拟 P2P 树状向各省市边缘节点扩散。最终返回的 `time` 用于评估全网推送的最小延迟，若返回 `-1` 则暴露出网络覆盖的物理盲区。

## 💻 最终 Java 代码实现

```java
import java.util.Queue;
import java.util.LinkedList;

class Solution {
    public int orangesRotting(int[][] grid) {
        // count 用于记录【新鲜橘子】的总数，也是我们 O(1) 终局审判的核心
        int count = 0;
        // time 用于记录耗费的分钟数，即 BFS 扩散的层数
        int time = 0;
        // queueSize 用于在 while 循环中控制按层（按分钟）遍历
        int queueSize = 0;
        // 核心数据结构：队列，用于存储【当前正在腐烂的橘子坐标】
        Queue<int[]> queue = new LinkedList<>();
        
        // 1. 【初始化阶段】：双重循环扫一遍网格，寻找超级源点
        for (int i = 0; i < grid.length; i++) {
            for (int j = 0; j < grid[0].length; j++) {
                if (grid[i][j] == 1) {
                    // 顺手统计新鲜橘子的数量
                    count++;
                }
                if (grid[i][j] == 2) {
                    // 发现初始的烂橘子！将它们全部入队，作为多源 BFS 的第一层“火种”
                    queue.offer(new int[]{i, j});
                }
            }
        }
        
        // 2. 【多源 BFS 扩散阶段】
        // 核心剪枝：不仅要求队列不为空，还要求新鲜橘子数 count > 0。
        // 如果 count 已经为 0，说明所有橘子都烂了，直接提前结束，避免 time 多加 1 分钟！
        while (!queue.isEmpty() && count > 0) {
            // 拿到当前队列的大小，这代表了【当前这一分钟】有多少烂橘子准备向外放毒
            queueSize = queue.size();
            
            // 必须严格使用初始的 queueSize 进行循环，保证只处理当前层的橘子
            for (int i = 0; i < queueSize; i++) {
                // 弹出一个当前层的烂橘子
                int[] cell = queue.poll();
                int x = cell[0];
                int y = cell[1];
                
                // --- 开始向四周扩散，并执行“原地修改” ---
                // 注：此处手动展开 4 个 if 能够规避 dirs 数组遍历带来的栈开销，实现 1ms 极限性能
                
                // 向上扩散：如果不越界，且上面是个新鲜橘子
                if (x > 0 && grid[x - 1][y] == 1) {
                    grid[x - 1][y] = 2; // 立刻让它腐烂（原地修改状态，防止重复入队）
                    queue.offer(new int[]{x - 1, y}); // 把新腐烂的橘子推入队列，参与下一分钟的扩散
                    count--; // 新鲜橘子数减 1
                }
                // 向下扩散
                if (x < grid.length - 1 && grid[x + 1][y] == 1) {
                    grid[x + 1][y] = 2;
                    queue.offer(new int[]{x + 1, y});
                    count--;
                }
                // 向左扩散
                if (y > 0 && grid[x][y - 1] == 1) {
                    grid[x][y - 1] = 2;
                    queue.offer(new int[]{x, y - 1});
                    count--;
                }
                // 向右扩散
                if (y < grid[0].length - 1 && grid[x][y + 1] == 1) {
                    grid[x][y + 1] = 2;
                    queue.offer(new int[]{x, y + 1});
                    count--;
                }
            }
            // 当前这批烂橘子全部向四周毒害了一圈，时间过去 1 分钟
            time++;
        }
        
        // 3. 【终局审判】
        // 如果 count == 0，说明成功感染所有橘子，返回耗时 time。
        // 如果 count 还不为 0，说明有橘子处于绝对安全岛中，永远烂不掉，返回 -1。
        return count == 0 ? time : -1;
    }
}
