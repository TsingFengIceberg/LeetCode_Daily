# 算法复盘笔记：LeetCode 20. 有效的括号 (Valid Parentheses)


---

## 💡 核心思路：后进先出 (LIFO) 的消除游戏
处理嵌套结构（如括号、HTML标签、JSON对象），**栈 (Stack)** 是唯一且最完美的数据结构。
整个匹配过程就像是一个“俄罗斯方块消除游戏”：
1. **左括号入栈**：遇到 `(`、`[`、`{`，说明开启了一个新的嵌套层级，直接把它们压入栈中等待未来的匹配。
2. **右括号出栈比对**：遇到 `)`、`]`、`}`，说明当前层级要闭合了。此时必须回头看栈顶元素（最近的一个左括号）是否和它是一对。
   * 如果栈是空的（说明右括号多了，没人跟它配对），直接 `false`。
   * 如果栈顶元素类型不匹配（比如 `(` 碰到了 `]`），直接 `false`。
3. **大结局清算**：字符串遍历结束后，如果栈里还有元素（说明左括号多了，没闭合完），返回 `false`；如果栈全空了，说明所有括号都完美牵手，返回 `true`。



---

## 💻 工业级代码与详尽注释 (ArrayDeque 性能进化版)

> **面试官的加分 Tip**：把 `java.util.Stack` 换成 `java.util.Deque` 的实现类 `ArrayDeque`。去掉了无用的同步锁，不仅能在面试中展现你对 Java 集合底层源码的深度理解，还能把耗时直接从 3ms 压缩到 1ms 甚至 0ms！

```java
import java.util.ArrayDeque;
import java.util.Deque;

class Solution {
    public boolean isValid(String s) {
        // 【大厂选型】使用 ArrayDeque 替代老旧且带锁的 Stack，大幅提升单线程性能
        Deque<Character> stack = new ArrayDeque<>();
        
        for (char c : s.toCharArray()) {
            // 遇到左括号，无脑压入栈中等待匹配
            if (c == '(' || c == '{' || c == '[') {
                stack.push(c);
            } else {
                // 边界防范 1：遇到右括号时，如果栈已经空了，说明右括号多余
                if (stack.isEmpty()) {
                    return false;
                }
                
                // 弹出栈顶的左括号进行比对
                char top = stack.pop();
                
                // 只要出现一对不匹配，整个字符串直接判死刑
                if ((c == ')' && top != '(') ||
                    (c == '}' && top != '{') ||
                    (c == ']' && top != '[')) {
                    return false;
                }
            }
        }
        
        // 边界防范 2：遍历结束后，如果栈没空，说明左括号多余没闭合
        return stack.isEmpty();
    }
}

```

### 📊 复杂度分析

* **时间复杂度**: $O(N)$。只需遍历一次字符串，入栈和出栈操作的时间复杂度均为 $O(1)$。
* **空间复杂度**: $O(N)$。最坏情况下（比如字符串全是左括号 `((((((`），所有的字符都会被压入栈中，占用与字符串长度相等的额外空间。

## 🏗️ 真实大厂工程实例深度剖析

1. **IntelliJ IDEA / VS Code 实时语法树构建 (AST)**：当你在 IDE 中编写代码时，后台极其轻量级的 Linter 线程会在你敲击键盘的瞬间，运行这套“栈匹配”逻辑。如果检测到 `}` 多余或者没有配对的 `(`，IDE 会立即通过行号映射，在你的代码下方画出红色波浪线报错。
2. **底层 JSON 报文解析器 (Jackson/Gson)**：微服务之间传输的巨型 JSON 字符串，本质上就是由 `{` 和 `[` 嵌套起来的。解析框架在反序列化之前，必须使用栈对整个报文进行词法扫描。只有当栈最终为空（嵌套合法），框架才敢分配内存去创建真正的 Java 对象，防止被恶意截断的残缺报文搞出内存泄漏。
3. **Chrome / Safari 浏览器渲染引擎**：前端的 HTML 标签（如 `<div>` 和 `</div>`）本质上就是“加了字母的括号”。浏览器的引擎通过栈结构来维护当前标签的嵌套深度，遇到闭合标签就出栈配对，最终以此构建出你在屏幕上看到的立体 DOM 树。
