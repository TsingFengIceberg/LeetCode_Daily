# [0169. 多数元素 (Majority Element)]

## 💡 初见端倪：固定间距对比法的陷阱与 $O(N \log N)$ 兜底方案
在最初拿到进阶要求时，我们尝试了一种极具直觉的解法：寻找相距 $N/2$ 且相同的两个数字。
**易错点纠正**：这种思路在“无序数组”中存在致命陷阱。例如在 `[2, 2, 1, 1, 2]` 中（$N/2 = 2$），索引 0 和 2、1 和 3、2 和 4 的数字均不相同，算法会直接失效。
**变废为宝（排序法）**：如果在这个思路前加上 `Arrays.sort(nums)` 进行预处理，由于多数元素数量 $> \lfloor N/2 \rfloor$，排序后它必定死死占据数组正中间 `nums[n/2]` 的位置。但代价是时间复杂度退化为 $O(N \log N)$，无法满足进阶要求。

## ⚔️ $O(N)$ 核心破局：摩尔投票法 (Boyer-Moore) 与“同归于尽”推演
为了达成 $O(N)$ 时间与 $O(1)$ 空间，我们引入了底层的**摩尔投票法（阵地火拼模型）**：
维护一个擂主 `boss` 和血量 `count`。遇到同阵营士兵则加血（`count++`），遇到不同阵营则一换一同归于尽（`count--`）。当血量归零时，更换新擂主。
**高光探讨（鹬蚌相争渔翁得利）**：
在推演时，我们深刻探讨了“不同少数派之间互相消耗”的情况。这正是摩尔投票法数学证明的最精妙之处：算法的最坏情况是“所有少数派联合起来消耗多数派”（即便如此多数派也会剩至少 1 人）。因此，少数派之间的内斗（鹬蚌相争），只会加速消耗敌军总数，让多数派（渔翁）的领先优势比例进一步扩大，这不仅不会破坏算法，反而让多数派赢得更稳！

## 🏭 工业界真实工程实例：极其苛刻环境下的空间 $O(1)$ 降维打击
在真实的高并发大厂架构中，摩尔投票法完美解决了 `HashMap` 统计频次带来的内存打爆（OOM）和 GC 停顿问题：
1. **硬件级 DDoS 流量清洗**：面对 100 Gbps 的恐怖流量，交换机极其昂贵的 SRAM 根本存不下 IP 访问哈希表。直接在芯片寄存器烧录摩尔投票逻辑，以 $O(1)$ 的空间瞬间清洗并揪出占据 $> 50\%$ 核心带宽的黑客恶意 IP。
2. **微服务雪崩的极速熔断**：当底层服务宕机引发网关“日志风暴”时，监控 Agent 若用哈希表全量聚合异常日志会直接被拖死。通过滑动时间窗口内的流式摩尔投票，0 内存分配即可在 500ms 内精准揪出宕机的下游微服务 ErrorCode，瞬间触发 Hystrix/Sentinel 熔断降级。

## 🎯 大厂面试 Follow-up 变形预警
如果顺利手撕本题，面试官极有可能抛出 **LeetCode 229. 多数元素 II**：
* **变形要求**：找出所有出现次数严格大于 $\lfloor N/3 \rfloor$ 的元素。
* **思路进阶**：数量 $> 1/3$ 的元素最多可能有 **2个**。因此我们需要在阵地上同时设立**两个擂台**（`boss1`, `boss2` 及对应的 `count1`, `count2`）。当遇到不属于这两者的第三方时，执行“三方大混战”，同时扣除两个擂主的血量。

## 最终 Java 代码实现
```java
class Solution {
    /**
     * 寻找多数元素 (Boyer-Moore 摩尔投票法)
     * 时间复杂度: O(N) - 遍历数组一次
     * 空间复杂度: O(1) - 仅使用两个 int 型变量，常量级额外空间
     */
    public int majorityElement(int[] nums) {
        // 初始化第一个数字为“擂主”（boss），并给他分配 1 点血量（count）
        int boss = nums[0];
        int count = 1;
        
        // 微操优化：因为 nums[0] 已经登台，直接从索引 1 开始遍历，减少一次冗余循环
        for (int i = 1; i < nums.length; i++) {
            if (nums[i] == boss) {
                // 遇到“自己人”，血量 +1
                count++;
            } else {
                // 遇到“敌人”，强行一换一同归于尽，血量 -1
                count--;
                // 阵地失守，刚刚把你干掉的“敌人”立刻占领阵地成为新擂主
                if (count == 0) {
                    boss = nums[i];
                    count = 1; // 新擂主上台，初始血量恢复为 1
                }
            }
        }
        // 题目保证多数派人数 > n/2，最后死死守在阵地上的必定是多数派
        return boss;
    }
}
