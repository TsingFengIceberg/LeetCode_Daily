# [0035. 搜索插入位置 (Search Insert Position)]

## 💡 算法灵魂：二分查找的“百年死角”与越界的美学
二分查找（Binary Search）号称是“思路最简单，Bug 最多”的算法。这道题的灵魂不在于“找到”，而在于**“找不到时，该停在哪里”**。
我们使用了最标准的**两端闭区间**模板（`left = 0`, `right = nums.length - 1`, `left <= right`）。



* **防溢出微操**：使用 `left + (right - left) / 2` 计算中间点，彻底封死了当数组极其巨大时 `left + right` 导致整型溢出（变成负数）的经典 Bug。
* **越界的美学 (`return left`)**：当 `while (left <= right)` 循环因为找不到目标值而被打破的瞬间，`left` 和 `right` 必然错开（即 `left = right + 1`）。由于我们遇到比 `target` 小的值就会执行 `left = mid + 1`，所以当循环结束时，`left` 指针刚好完美地越过了所有小于 `target` 的元素，稳稳地停在了**第一个大于 `target` 的位置**上。而这，恰好就是物理意义上最完美的插入点！

## 🏢 工业界真实工程实例：跨越海量数据的时空定位
这种 $O(\log n)$ 的“下界查找 (Lower Bound)”算法，是所有海量数据系统的底层生命线：
1. **MySQL InnoDB 的 B+ 树页内查找**：当向数据库插入一条新记录（如 `ID = 5`）时，引擎会在 16KB 的数据页（内部是由主键升序排列的数组）中，直接跑这套二分代码。找不到 5 没关系，返回的 `left` 指针就是内存中需要被撑开、以便将 5 严丝合缝插进去的精确物理偏移量。
2. **Prometheus / Kafka 时间序列定位**：在成百上千亿条按时间戳递增排列的监控日志中，寻找 `2026-03-12 14:00:00` 之后的报错。如果这一秒刚好没日志，系统依赖 `return left` 瞬间定位到“这一秒之后的最近一条未来日志”，绝不遗漏任何故障现场。
3. **爱奇艺视频关键帧 (I-Frame) 极速跳转**：当你把进度条拖拽到 83 秒处，播放器底层拿着 83 去关键帧时间数组里做二分查找。返回的插入位置，能帮引擎瞬间锚定最近的独立解码帧，实现丝滑的拖拽播放。

## 🧮 复杂度剖析 (降维打击)
* **时间复杂度：$O(\log n)$**。对比 $O(n)$ 的线性遍历，这是马车与曲率飞船的差距。如果数组有 40 亿个元素，暴力遍历最差要找 40 亿次，而二分查找最差只需要找 **32 次**！
* **空间复杂度：$O(1)$**。全程只维护了三个整型指针（`left`, `right`, `mid`），没有任何额外的内存开销。

## 💻 最终 Java 代码实现 (0ms 大厂极致闭区间版)

```java
class Solution {
    public int searchInsert(int[] nums, int target) {
        // 1. 设置两端闭区间的起跑线
        int left = 0;
        int right = nums.length - 1;
        
        // 2. 循环生死线：因为是闭区间，left 和 right 重合时依然需要检查
        while (left <= right) {
            // 🚨 防溢出微操：绝不写 (left + right) / 2
            int mid = left + (right - left) / 2;
            
            if (nums[mid] == target) {
                // 运气好，直接命中目标值
                return mid;
            } else if (nums[mid] < target) {
                // 目标值在右侧，收缩左边界
                left = mid + 1;
            } else {
                // 目标值在左侧，收缩右边界
                right = mid - 1;
            }
        }
        
        // 3. 越界的美学：当循环结束，left 刚好停在第一个大于 target 的位置（即插入点）
        return left;
    }
}
