# 📘 LeetCode 75. 颜色分类 (Sort Colors) - 面试复盘笔记

## 1. 题目背景与核心思想
* **历史渊源**：本题史称“荷兰国旗问题 (Dutch National Flag Problem)”，由图灵奖得主 Dijkstra 提出。
* **直觉解法 (两次遍历)**：计数排序。第一遍统计 0、1、2 的个数，第二遍按数量覆盖原数组。时间 $O(N)$，空间 $O(1)$，极其稳定但缺乏技术深度。
* **大厂进阶解法 (一趟扫描)**：利用**三路划分 (3-Way Partitioning)** 和三指针向中间挤压。不仅要求 $O(1)$ 空间，更要求在流式数据（不可逆回溯）场景下一次性完成分类。



## 2. 🌟 核心死角剖析：为什么遇到 2 时，游标 i 绝对不能加 1？

这是大厂面试这道题时**最核心的拷问点**。我们需要从“循环不变量 (Loop Invariant)”和“区域状态机”的角度来绝对严谨地推导。

在循环的任意时刻，三个指针 `l`, `i`, `r` 将数组严格划分成了 4 个物理区域：
* **区域 A `[0, l]`**：全都是确认过的 `0` (红色)。
* **区域 B `[l+1, i-1]`**：全都是确认过的 `1` (白色)。注意，`i` 左边的元素都是我们探索过且处理妥当的。
* **区域 C `[i, r-1]`**：**未知探索区（盲盒区）！** 可能是 0、1、2 中的任意一个。
* **区域 D `[r, nums.length - 1]`**：全都是确认过的 `2` (蓝色)。

**指针移动的生死细节对比：**
* **遇到 0，为什么 `i` 可以 `++`？**
  当我们把 `nums[i]` 上的 0 与 `nums[l+1]` 交换时，`l+1` 是区域 B 的边界。也就是说，换到 `i` 位置上的那个元素，**百分之百是一个已经被确认过的 `1`**（或者就是 `i` 本身）。既然换过来的数字处理妥当了，探索者 `i` 就可以安心地向前走一步（`i++`）。
* **遇到 2，为什么 `i` 绝对不能 `++`？**
  当我们把 `nums[i]` 上的 2 与 `nums[r-1]` 交换时，`r-1` 位于区域 C（未知盲盒区）的最右侧！
  你把右边的一个“盲盒”强行塞到了 `i` 位置。**这个新换过来的数字完全是个未知数（可能是 0，也可能是 1 或 2）**。如果此时执行 `i++`，探索者就会直接跨过并遗漏这个盲盒，导致排序崩溃。
  因此，**必须把 `i` 停在原地**，在下一轮循环中，亲自“开盲盒”检验这个刚从右边换过来的数字！

## 3. Java 工程演进与代码实现

### 🟢 方法一：两次遍历计数法 (O(N) 时间，稳妥工程解)
```java
public class Solution1 {
    public void sortColors(int[] nums) {
        if (nums == null || nums.length <= 1) return;
        int count0 = 0, count1 = 0, count2 = 0;
        for (int num : nums) {
            if (num == 0) count0++;
            else if (num == 1) count1++;
            else if (num == 2) count2++;
        }
        int i = 0;
        while (count0-- > 0) nums[i++] = 0;
        while (count1-- > 0) nums[i++] = 1;
        while (count2-- > 0) nums[i++] = 2;
    }
}

```

### 🚀 方法二：一趟扫描三指针法 (0ms 满分，击败 100%)

极度优雅的边界控制（`l = -1`, `r = nums.length`），避免了大量特判逻辑。

```java
public class Solution2 {
    public void sortColors(int[] nums) {
        // l: 0的右边界；r: 2的左边界；i: 探索游标
        int l = -1, r = nums.length, i = 0;
        int temp;
        
        while (i < r) { // 当游标撞到 2 的区域时，探索结束
            if (nums[i] == 0) {
                // 遇到 0，将其与 1 区的左边界交换
                temp = nums[i];
                nums[i] = nums[l + 1];
                nums[l + 1] = temp;
                l++; 
                i++; // 换过来的必然是 1，可以放心前进
            } else if (nums[i] == 2) {
                // 遇到 2，将其与盲盒区最右侧交换
                temp = nums[i];
                nums[i] = nums[r - 1];
                nums[r - 1] = temp;
                r--; 
                // 【严禁 i++】：换过来的是右侧盲盒元素，下一轮需原地再判断！
            } else {
                // 遇到 1，放行
                i++;
            }
        }
    }
}

```

## 4. 大厂真实业务映射

这套“一趟扫描、三路划分”的原地算法，在底层基础架构中是极其核心的优化手段：

* **`Arrays.sort()` 底层源码优化**：双轴快排（Dual-Pivot Quicksort）利用三路划分应对包含大量重复元素的数组，将时间复杂度从退化的 $O(N^2)$ 强行拉回 $O(N \log N)$。
* **JVM 垃圾回收 (V8/CMS/G1)**：并发垃圾收集的“三色标记法 (Tri-color Marking)”，在 OOM 边缘极度缺内存的情况下，原地利用对象标记（白、灰、黑）进行零内存分配的扫描。
* **RTOS 实时操作系统调度**：在禁用动态内存分配的车机系统底层，利用原地三指针交换，在一趟扫描内将中断任务缓冲区按“高、中、低”优先级瞬间重排。
