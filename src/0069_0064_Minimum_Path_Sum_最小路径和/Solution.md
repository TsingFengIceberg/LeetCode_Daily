# [0064. 最小路径和 (Minimum Path Sum)]

## 💡 核心直觉：原地修改的神级降维打击
作为【62. 不同路径】的孪生兄弟题，这道题将“求路径总数”变成了“求路径权重的最小总和”。因为机器人只能向下或向右移动，到达任意格子 `grid[i][j]` 的最小路径和，必然来自于它上方格子和左方格子的较小值，加上它自身的权重。
在这一次挑战中，你展现出了极其敏锐的空间嗅觉，没有像常规思路那样去 `new` 一个二维或一维的 `dp` 数组，而是直接**利用原数组 `grid` 进行原地修改（In-place Modification）**，在遍历的过程中直接将历史最小权重和覆盖到原矩阵上，极其优雅地将额外空间复杂度强行压缩到了 **$O(1)$**！

## 🚨 Code Review 与弯路复盘：画蛇添足的边界判断
虽然你极其迅速地“秒杀”了这道题，但作为极其严苛的大厂 Code Review，必须一字不落地记录下你代码中“画蛇添足”的小瑕疵。
在你最初提交的代码中，在完成第一行和第一列的累加初始化后，你写了这样两段判断：
> `if (grid.length == 1) { return grid[0][grid[0].length - 1]; }`
> `if (grid[0].length == 1) { return grid[grid.length - 1][0]; }`

**为什么说这是多余的弯路？**
因为在你代码的开头，`for` 循环已经完美地把第一行和第一列的路径和全部累加完毕了。如果传入的矩阵真的只有一行（`grid.length == 1`），那么后续的嵌套 `for (int i = 1; ...)` 根本就不会触发执行。程序会非常安全地直接走到最后一行的 `return grid[grid.length - 1][grid[0].length - 1];` 并返回正确答案。
去掉这两处冗余的 `if`，代码不仅 100% 正确，而且会更加紧凑、纯粹，这才是工业界追求的“如无必要，勿增实体”。

## ⚙️ 工业界真实工程实例：计算机视觉与智能仓储
这种 $O(1)$ 原地修改的网格 DP 算法，在普通 CRUD 开发中极少见，但在底层图形引擎和高并发硬件调度中却是真正的“大杀器”：

1. **Photoshop 核心黑科技：Seam Carving (内容感知缩放)**
   当你在 PS 中缩小一张图片且不想让人脸被“挤扁”时，图形引擎会生成一张记录了图像特征的“能量矩阵”（类似于本题的 `grid`，人脸边缘能量高，白墙能量低）。系统需要找到一条从上到下**能量总和最小的路径（Seam）**并将其裁掉。由于 4K 图像的像素矩阵极其巨大，绝不能每次裁剪都开辟新的二维 DP 数组（会导致显卡显存瞬间 OOM），因此直接在原图的能量矩阵上做 $O(1)$ 原地累加修改，是 OpenGL/Vulkan 等底层渲染引擎的基操。

2. **京东/阿里菜鸟智能仓储：AGV 机器人局部动态避障**
   在现代巨型物流仓中，地面被划分为巨大的网格阵列，网格的权重代表“拥堵程度”或“地面摩擦耗电量”。后台 RCS（机器人控制系统）每秒要为上千台 AGV 重新规划成本最低的路线。这种极简的 `Math.min(grid[i - 1][j], grid[i][j - 1])` 原地 DP 算法，凭借极少的 CPU 指令周期和对 CPU Cache 极其友好的连续内存访问机制，成为了高频动态路径重规划的性能杀手锏。

## 💻 最终 Java 代码实现 (大厂满分级精简版)
* **时间复杂度**：$O(m \times n)$，必须遍历整张网格。
* **空间复杂度**：$O(1)$，直接复用输入矩阵 `grid`，无需任何额外空间！

```java
class Solution {
    // 【大厂满分级 Code Review】
    // 剔除了冗余的单行/单列 if 判断，逻辑更加纯粹凝练
    public int minPathSum(int[][] grid) {
        // 【边界初始化 1】第一行：由于只能从左往右走，当前格子的最小路径和就是它左边累加的结果加上它自己
        for (int i = 1; i < grid[0].length; i++) {
            grid[0][i] += grid[0][i - 1];
        }
        
        // 【边界初始化 2】第一列：由于只能从上往下走，当前格子的最小路径和就是它上面累加的结果加上它自己
        for (int i = 1; i < grid.length; i++) {
            grid[i][0] += grid[i - 1][0];
        }
        
        // 【核心状态转移】
        // 从 (1, 1) 开始遍历，每个格子取其上方和左方的最小值，累加到当前格子自身
        // 利用 O(1) 的原地修改，完美避免了二维 dp 数组的内存开销
        for (int i = 1; i < grid.length; i++) {
            for (int j = 1; j < grid[0].length; j++) {
                grid[i][j] += Math.min(grid[i - 1][j], grid[i][j - 1]);
            }
        }
        
        // 最终右下角的值即为全局最小路径和
        return grid[grid.length - 1][grid[0].length - 1];
    }
}
