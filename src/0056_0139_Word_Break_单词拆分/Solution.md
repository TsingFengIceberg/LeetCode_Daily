# [0139. 单词拆分 (Word Break)]

## 🧠 面试现场推导实录：如何一步步逼近 DP 真相？
在面对字符串切割问题时，我们摒弃了极易超时的回溯搜索（DFS），转而采用“接力赛”思维，通过三个核心问题，推导出了这套完美的动态规划解法：

1. **Java 性能微操（$O(N) \to O(1)$）**：
   * **直觉**：在频繁的内层循环中，必须极速判断单词是否在字典中。
   * **决策**：果断抛弃题目给定的 `List<String>`，在算法起手式直接将其转化为 `HashSet`。这一个动作，直接拯救了成千上万次的遍历开销。
2. **DP 的基石与发令枪（边界条件）**：
   * **直觉**：空字符串的处理逻辑。
   * **决策**：设定 `dp[0] = true`。这在逻辑上代表“长度为 0 的字符串天然合法”，在工程上，它是整场接力赛的**发令枪**。如果没有 `dp[0] = true`，第一棒 `dp[j]` 永远为 `false`，后续的递推将全部瘫痪。
3. **接力赛交接逻辑（核心状态转移方程）**：
   * **直觉**：当前切出来的碎片在字典里就行了。
   * **深度纠偏**：仅仅“碎片在字典里”不够，必须保证“前一个中转站”也是通的！
   * **最终方程落地**：`dp[j] && wordDictSet.contains(s.substring(j, i))`。并且一旦找到一个合法的 `j`，立刻 `break` 内层循环进行极速剪枝。

## 🏢 工业界真实工程实例：让计算机读懂人类语言
在 LeetCode 里这只是个布尔值判断，但在工业界，这套 DP 逻辑是自然语言处理（NLP）的底层基石：
1. **中文分词引擎 (Tokenization)**：中文没有空格，百度/谷歌在处理用户搜索（如“南京市长江大桥”）时，底层跑的就是带有超大词典的 DP 拆分算法。判断合法切分路径后，再结合隐马尔可夫模型（HMM）选出最符合人类语义的搜索关键词。
2. **社交平台 Hashtag 解析与广告画像**：小红书或 Twitter 每天有海量无空格标签（如 `#throwbackthursday`）。数据清洗中台通过此 DP 算法将其精准拆解为 `throwback` 和 `thursday`，为用户打上复古特征标签，从而实现精准的广告分发。
3. **极限性能进化 (Trie 树)**：面对百万级词典，大厂会将 `HashSet` 升级为**字典树 (Trie)**，通过树节点遍历取代极其消耗 CPU 和内存的 `substring` 字符串截取操作，实现性能的二次降维打击。

## 🧮 复杂度剖析
* **时间复杂度：$O(n^3)$**。其中 $n$ 是字符串 `s` 的长度。外层循环 $n$ 次，内层循环最多 $n$ 次，在内层循环中调用 `substring` 生成字符串并计算哈希值又需要花费 $O(n)$ 的时间。总时间复杂度为 $O(n^3)$。
* **空间复杂度：$O(n + m \times k)$**。其中 $n$ 是 `dp` 数组的长度，$m$ 是字典中单词的个数，$k$ 是单词的平均长度（用于存储 `HashSet`）。

## 💻 最终 Java 代码实现 (大厂接力赛剪枝版)

```java
import java.util.HashSet;
import java.util.List;
import java.util.Set;

class Solution {
    public boolean wordBreak(String s, List<String> wordDict) {
        // 1. 将 List 转换为 HashSet，实现 O(1) 极速查找
        Set<String> wordDictSet = new HashSet<>(wordDict);
        
        // 2. 初始化 dp 数组，长度为 s.length() + 1
        // dp[i] 表示字符串 s 的前 i 个字符能否被成功拆分
        boolean[] dp = new boolean[s.length() + 1];
        
        // 3. 埋下发令枪（边界条件）
        dp[0] = true;
        
        // 4. 外层循环 i：代表当前接力赛要跑到的终点（从 1 跑到 s.length()）
        for (int i = 1; i <= s.length(); i++) {
            // 内层循环 j：代表前一个中转站（从起跑线 0 开始，一直试到 i-1）
            for (int j = 0; j < i; j++) {
                
                // 🚨 【核心状态转移方程】
                // 必须同时满足：1. 之前的半截 (到 j) 能成功拆分；2. 剩下的半截 (j 到 i) 在字典中
                if (dp[j] && wordDictSet.contains(s.substring(j, i))) {
                    dp[i] = true;
                    break; // 只要找到一种合法的拆法，立刻停止寻找下一个 j，极速剪枝！
                }
            }
        }
        
        // 5. 最终返回：跑到整个字符串最后一位时，交接棒是否还在手里
        return dp[s.length()];
    }
}
