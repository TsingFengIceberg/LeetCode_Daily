# [0022. 括号生成 (Generate Parentheses)]

## 💡 算法灵魂：规则前置的动态剪枝 (合法状态机)
与全排列和子集的“无脑穷举”不同，括号生成是一台极其严谨的**嵌套状态机**。如果不加限制地拼接 `(` 和 `)`，会产生海量的非法废片（如 `)(())(`）。大厂的标准做法是：**在做选择的瞬间，就把不合法分支掐死在摇篮里**。



1. **左括号的包容性 (`open > 0`)**：只要手里还有左括号，随时可以打出去。
2. **右括号的生死线 (`open < close`)**：全题唯一的灵魂！`open` 和 `close` 代表的是**剩余量**。只有当剩余的右括号多于左括号时，才说明前面有“已打出但未匹配的左括号”在等我们，此时打出右括号才是合法的。
3. **隐式回溯魔法 (`String` 拼接)**：使用 `current + "("` 作为参数传递时，Java 会在底层生成一个全新的 String 对象。这导致当前层的 `current` 不会被污染，因此**不需要写撤销选择的代码**，逻辑极其清爽。

## 🚀 架构师深潜：可读性 vs 极限性能 的 Trade-off
本次提交执行用时为 2ms（击败 27.80%）。为什么不是 0ms？
* **性能损耗点**：由于 Java 中 `String` 的不可变性，每一次 `+` 拼接都会在堆内存中 `new` 出一个新对象。在 $n=8$ 时，会产生数万个极短命的垃圾对象，引发轻微的 GC（垃圾回收）停顿。
* **0ms 的代价**：如果想冲刺 0ms，必须换成全局唯一的 `StringBuilder`，并在每次递归后手动调用 `deleteCharAt()` 进行显式回溯。
* **架构取舍**：在 $n \le 8$ 的小数据规模下，2ms 完全可以接受。用极小的性能代价，换取了代码的极高可读性和“免撤销”的防错性，这在真实的团队协作中往往是更受推崇的代码风格。

## 🏢 工业界真实工程实例：校验 vs 生成的哲学
面试中常考的两大括号难题，背后是两种截然不同的系统设计哲学：
* **第 20 题“有效的括号”（校验）**：给你结果判对错，用**栈 (Stack)**。这代表了“先污染后治理”的低级防线。
* **本题“括号生成”（生成）**：从头造出所有合法组合，用**带状态计数器的回溯剪枝**。这代表了“规则前置”的高级系统设计。
在真实工业界，这种引擎被广泛用于：
1. **编译器测试 (Fuzzing)**：V8 引擎的模糊测试，利用此逻辑源源不断生成极度深层嵌套但语法完全合法的 JS 代码，去轰炸编译器的解析极限。
2. **前端 SSR 服务端渲染**：Vue/React 在服务端动态拼接 HTML 字符串时，利用此状态机保证 `<div>` 和 `</div>` 标签的 100% 完美闭合。
3. **ORM 框架的动态 SQL**：MyBatis 在动态拼装 `AND (A OR B)` 这种极其复杂的嵌套条件时，从代码层掐死 SQL 括号不匹配的语法灾难。

## 🧮 复杂度剖析
* **时间复杂度：$O(\frac{4^n}{\sqrt{n}})$**。这是组合数学中著名的**卡特兰数 (Catalan Number)**。由于我们极其精准地剪去了所有非法分支，算法生成的就是第 $n$ 个卡特兰数个组合。
* **空间复杂度：$O(n)$**。除去保存结果的 `List` 不计，递归调用栈的最大深度为 $2n$（打光所有左右括号）。

## 💻 最终 Java 代码实现 (大厂高可读性剪枝版)

```java
import java.util.ArrayList;
import java.util.List;

class Solution {
    private List<String> result = new ArrayList<>();
    
    public List<String> generateParenthesis(int n) {
        // 传入初始空字符串，以及手里握着的左括号数量 n、右括号数量 n、和最大对数 n
        backtrack(new String(), n, n, n);
        return result;
    }

    private void backtrack(String current, int open, int close, int max) {
        // 1. 杀青快照：长度等于总括号数时，收集结果
        if (current.length() == max * 2) {
            result.add(current);
            return;
        }
        
        // 2. 打出左括号：只要手里还有，随便打
        // 隐式回溯：current + "(" 生成新对象，不污染本层状态
        if (open > 0) {
            backtrack(current + "(", open - 1, close, max);
        }
        
        // 3. 🚨 灵魂生死线：只有当剩余右括号 > 剩余左括号时，才能打出右括号！
        if (open < close) {
            backtrack(current + ")", open, close - 1, max);
        }
    }
}
