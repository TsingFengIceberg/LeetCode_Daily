# 📘 LeetCode 55. 跳跃游戏 (Jump Game) - 面试复盘笔记

## 1. 题目核心痛点与思维误区
* **直觉陷阱**：遇到“能否到达”的问题，初学者极易陷入 DFS/BFS（回溯穷举）或 $O(n^2)$ 的动态规划陷阱，导致在海量数据下必定超时。
* **破局核心（贪心算法）**：我们根本不需要知道具体的“跳跃路径”，只需要关心在不断的跳跃过程中，我们所能覆盖的**“最远可达边界 (maxReach)”**是多少即可。



## 2. 核心状态转移逻辑
维护一个全局变量 `maxReach`，表示目前能够安全抵达的最远下标。遍历数组中的每一个位置 `i`：
1. **死亡判定（极其关键）**：如果你当前走到的下标 `i` 已经超过了历史积攒的 `maxReach`（即 `i > maxReach`），说明你遇到了“断崖”（比如遇到了 0 且无法跨越），彻底卡死，直接返回 `false`。
2. **状态更新**：如果当前节点可达，尝试用当前节点的能力拓宽边界：`maxReach = Math.max(maxReach, i + nums[i])`。
3. **提前剪枝（性能优化）**：一旦发现 `maxReach` 已经能够覆盖甚至超越终点下标（`nums.length - 1`），立刻停止后续遍历，直接返回 `true`。

## 3. Java 核心代码实现 (满分贪心版)
* **Time Complexity**: $O(n)$，只需从头到尾遍历数组一次。
* **Space Complexity**: $O(1)$，只使用了一个整形变量，内存极其轻量。

```java
public class Solution {
    public boolean canJump(int[] nums) {
        int maxReach = 0;
        for (int i = 0; i < nums.length; i++) {
            // 死亡判定：如果当前位置已经超出了能到达的最远边界，说明彻底断档
            if (i > maxReach) {
                break;
            }
            // 贪心更新：当前位置能触达的最远边界，和历史最远边界打擂台
            maxReach = Math.max(maxReach, i + nums[i]);
            // 提前剪枝：只要够得着终点，立刻返回，不拖泥带水
            if (maxReach >= nums.length - 1) {
                return true;
            }
        }
        return false;
    }
}

```

## 4. 大厂真实业务映射

这套极简的 $O(n)$ 贪心预测逻辑，在底层的系统架构中被广泛应用于**资源覆盖率预判**：

* **流媒体播放器（字节/B站）**：预测预加载缓冲队列（Buffer）是否足以覆盖用户未来的播放时间，防止卡顿。
* **无线传感器路由（阿里/腾讯）**：极速评估网络数据包顺着中继节点进行 Relay 广播时，TTL（存活跳数）能否支撑其到达终端。
* **无人配送续航（美团/京东）**：评估沿途换电站的补给能力能否支撑无人车跑完全程。
