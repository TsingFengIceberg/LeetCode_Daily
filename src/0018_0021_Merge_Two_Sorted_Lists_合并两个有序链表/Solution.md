# 算法复盘笔记：LeetCode 21. 合并两个有序链表 (Merge Two Sorted Lists)

## 一、 题目剖析与大厂考点
* **考察频次**：⭐⭐⭐⭐⭐ (所有大厂的白板热身题，极为高频)
* **核心标签**：链表、双指针、递归
* **时间/空间复杂度要求**：面试官终极期望为 Time: $O(N+M)$, Space: $O(1)$（迭代法）。
* **题目痛点**：
  1. 链表头节点的初始化极其繁琐，容易写出臃肿的 `if-else`。
  2. 遍历结束后的收尾工作，新手容易用 `while` 循环一个个接，暴露基本功不足。
* **大厂延伸**：本题是归并排序（Merge Sort）的底层核心，更是高频 Hard 题“23. 合并 K 个升序链表”的基石。

## 二、 核心解法沉淀

### 方法 1：迭代法 + 伪头节点 (Dummy Node) (大厂首选，极力推荐)
利用一个无意义的 `Dummy Node` 站岗，彻底消灭头节点为空的边界判断。利用链表的物理连贯性，在 $O(1)$ 时间内完成最后剩余部分的拼接。
* **复杂度**：时间 $O(N+M)$，空间 $O(1)$。
* **架构师视角的优势**：极其稳定，无论链表多长都不会导致内存溢出，是高并发工业级代码的绝对首选。

**核心 Java 代码定稿 (0ms 击败 100%)**：
```java
public class Solution {
    public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
        // 伪头节点，消除边界判断
        ListNode dummy = new ListNode(-1);
        ListNode curr = dummy;
        
        // 比较并穿针引线
        while (l1 != null && l2 != null) {
            if (l1.val <= l2.val) {
                curr.next = l1;
                l1 = l1.next;
            } else {
                curr.next = l2;
                l2 = l2.next;
            }
            curr = curr.next;
        }
        
        // O(1) 瞬间拼接剩余部分
        curr.next = (l1 != null) ? l1 : l2;
        
        return dummy.next;
    }
}

```

### 方法 2：递归法 (大厂装杯神技)

纯粹基于数学归纳法：谁小，谁就是新链表的头，其 `next` 指向剩下的两段链表继续合并的结果。

* **复杂度**：时间 $O(N+M)$，空间 $O(N+M)$ (隐式调用栈)。
* **工业界避坑指南**：代码极其优美，但在海量数据的真实业务中**绝对禁用**。因为递归深度正比于链表总长度，极易触发 JVM 的 `StackOverflowError` 导致进程直接崩溃。但在面试白板上写出，能展现出强大的抽象思维。

## 三、 工业界真实场景映射 (工程实例)

1. **LSM-Tree 底层 Compaction (数据合并)**：在 TiDB、HBase、RocksDB 等分布式数据库中，内存数据刷入磁盘形成多个有序文件 (SSTable)。后台线程会不断将这些小的有序文件合并成大的有序文件，其底层的指针比对逻辑与本题完全一致。
2. **分布式搜索引擎的 Scatter-Gather (分散-聚集) 模式**：在 Elasticsearch 中，网关节点收到各分片发回的局部有序结果后，利用多路归并的思想整合出全局有序结果，避免了高昂的全局重排序计算开销。
