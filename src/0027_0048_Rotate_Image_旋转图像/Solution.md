## 💡 核心思路：按层模拟与四点循环交换
本题的硬性要求是 **原地 (in-place)** 修改，这意味着不能开辟新的二维数组（空间复杂度必须为 `O(1)`）。
最佳实践是采取“剥洋葱”的按层推进法，由外向内，每次处理一层（一个正方形环）。在当前层中，按顺时针方向，四个元素为一组进行循环交换。



### 🚨 避坑指南与易错点分析
1. **致命的“Off-by-One”越界**：
   在长度为 `n` 的矩阵中，寻找对称对角坐标时，必须使用 `n - 1 - index`，绝不能错写为 `n - index`。
2. **内层循环的边界控制（防重复旋转）**：
   每一层有 `n - 1 - 2 * i` 个元素需要作为起点交换。内层循环变量 `j` 的范围必须是 `i <= j < n - 1 - i`（**严格小于**右边界），否则会导致四个角被重复反转，逻辑全毁。
3. **坐标推导映射关系**：
   假设当前处于第 `i` 层，第 `j` 列，顺时针的四个坐标严格推导如下：
   - 顶部：`matrix[i][j]`
   - 右侧：`matrix[j][n - 1 - i]`
   - 底部：`matrix[n - 1 - i][n - 1 - j]`
   - 左侧：`matrix[n - 1 - j][i]`

## 💻 Java 最终定稿代码

```java
class Solution {
    public void rotate(int[][] matrix) {
        int n = matrix.length;
        
        // 外层循环：控制层数，i 表示当前层（从最外层 0 开始，最多到 n/2 层）
        for (int i = 0; i < n / 2; i++) {
            // 内层循环：控制当前层需要作为起点交换的元素
            // 注意边界是严格小于 n - 1 - i，把最后一个元素留给下一条边的起点
            for (int j = i; j < n - 1 - i; j++) {
                
                // 1. 备份顶部元素（左上）
                int temp = matrix[i][j];
                
                // 2. 左侧元素移动到顶部 (Left -> Top)
                matrix[i][j] = matrix[n - 1 - j][i];
                
                // 3. 底部元素移动到左侧 (Bottom -> Left)
                matrix[n - 1 - j][i] = matrix[n - 1 - i][n - 1 - j];
                
                // 4. 右侧元素移动到底部 (Right -> Bottom)
                matrix[n - 1 - i][n - 1 - j] = matrix[j][n - 1 - i];
                
                // 5. 顶部备份元素移动到右侧 (Temp -> Right)
                matrix[j][n - 1 - i] = temp;
            }
        }
    }
}

```

## 📊 复杂度分析

* **时间复杂度 (Time Complexity)**：`O(N^2)`，其中 N 为矩阵边长。矩阵中的每个元素仅被访问和移动了一次。
* **空间复杂度 (Space Complexity)**：`O(1)`。仅使用了常数级别的额外变量（`n`, `i`, `j`, `temp`），完全满足原地修改的要求。

## 🚀 大厂实战与思维拓展 (Follow-up)

在字节跳动、腾讯等一线大厂的面试中，这道题经常会作为“手撕代码”的第一题，以此来考察候选人的代码控制力和 Debug 能力。完成本题后，面试官通常会抛出以下 Follow-up：

1. **业务外衣变种**：
* **场景**：“假设你在做一个图片编辑 App 的服务端，现在用户不仅要求旋转，还要求顺时针旋转后进行镜像对称翻转（或者逆时针旋转 90 度），你怎么做？”
* **解答**：这时候如果去硬推四点坐标就容易乱。可以引入**“翻转组合法”**。顺时针 90 度 = 先上下水平翻转，再沿主对角线翻转。逆时针 90 度 = 先左右垂直翻转，再沿主对角线翻转。代码极其干净。


2. **极端场景 Follow-up**：
* **提问**：“如果这是一个 `M x N` 的长方形矩阵，还能做到 `O(1)` 空间复杂度原地旋转吗？”
* **解答**：非常困难/几乎不可能在面试时间内写出完美的原地解。此时必须明确告知面试官：长宽不等时，原地旋转涉及到复杂的循环移位（Cycle Leader Algorithm），在真实工程中，为了可维护性和绝对性能，通常会妥协使用 `O(M * N)` 的额外空间直接映射，或者采用我们在工程课里讲的“分块读取”机制。
