# [0118. 杨辉三角 (Pascal's Triangle)]

## 💡 算法灵魂：二维动态规划 (2D DP) 的完美起手式
“杨辉三角”是所有二维矩阵路径问题、背包问题以及复杂 DP 的前置教科书。
* **状态定义**：我们可以将其看作一个左对齐的直角三角形矩阵。第 `i` 行的第 `j` 个元素可以定义为状态 $dp[i][j]$。
* **边界条件 (Base Case)**：每一行的开头（$j=0$）和结尾（$j=i$）永远是 1。这相当于我们 DP 矩阵的“墙壁”，不需要任何计算。
* **状态转移方程**：对于处于中间的元素，它极其严格地依赖于上一层的两个状态：正上方和左上方。
  即：$dp[i][j] = dp[i-1][j-1] + dp[i-1][j]$。
掌握了这个推导公式，后续所有的矩阵寻路问题对你来说都将只是变种。

## 🏢 工业界真实工程实例：量化金融与流量潮汐
在代码之外，杨辉三角本质上是组合数学 $C(n, k)$ 和二项式分布的展现，在顶尖工业界有着极高价值的应用：
1. **量化金融的“二项式期权定价模型” (BOPM)**：华尔街在为复杂的衍生品定价时，会将未来股价的涨跌模拟成一棵二叉树。在离散的时间步长后，股价到达某一特定金额的“路径总数”，完美契合杨辉三角的数字分布。结合涨跌概率，这就是期权定价的底层核心引擎。
2. **复杂 A/B Testing 与流量漏斗 (高尔顿板原理)**：在微服务架构的多层级路由灰度发布中，假设请求经过多层 50/50 概率的随机网关分发，最终落在底层各服务器集群的流量比例，精确符合杨辉三角的底层分布。架构师据此预估“中间节点”将承受的流量洪峰，提前完成机器扩容。

## 🧮 复杂度剖析
* **时间复杂度：$O(numRows^2)$**。因为我们需要计算的总元素个数大约是 $1 + 2 + 3 + \dots + numRows$，这是一个等差数列求和，量级是平方级别的。
* **空间复杂度：$O(numRows^2)$**。我们需要创建一个嵌套的二维 `List` 来存储所有的生成结果并返回，总共存储了与时间复杂度同等数量的元素。（注：如果不算返回结果占用的空间，算法本身的额外空间为 $O(1)$）。

## 💻 最终 Java 代码实现 (大厂标准微操版)

```java
import java.util.ArrayList;
import java.util.List;

class Solution {
    public List<List<Integer>> generate(int numRows) {
        // 全局结果集
        List<List<Integer>> result = new ArrayList<>();

        for (int i = 0; i < numRows; i++) {
            // 当前行的容器
            List<Integer> row = new ArrayList<>();
            // 1. 绝对边界：每一行的第一个元素必定是 1
            row.add(1); 

            // 2. 状态转移：计算中间的元素 (依赖于上一行)
            // 巧妙的循环条件：当 i=0 时进不去循环，当 i=1 时也进不去，完美避开越界
            for (int j = 1; j < i; j++) {
                // dp[i][j] = dp[i-1][j-1] + dp[i-1][j]
                int sum = result.get(i - 1).get(j - 1) + result.get(i - 1).get(j);
                row.add(sum);
            }
            
            // 3. 绝对边界：只要不是第 0 行，每一行的最后一个元素必定是 1
            if (i > 0) {
                row.add(1); 
            }
            
            // 将拼装好的当前行加入结果集
            result.add(row);
        }

        return result;
    }
}
